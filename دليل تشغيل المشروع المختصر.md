# دليل تشغيل المشروع المختصر

هذا الدليل يشرح طريقة تشغيل المشروع محليا خطوة بخطوة، مع أشهر المشاكل التي قد تظهر أثناء التشغيل وطريقة حلها.

## فكرة مهمة قبل التشغيل

المشروع مقسم إلى 3 أجزاء:

| الجزء | المجلد | الوظيفة |
|---|---|---|
| Backend | `backend` | Laravel API وقاعدة البيانات |
| Website | `موقع اتعلم بل العربي` | واجهة الموقع الرئيسية |
| Admin Dashboard | `لوحه التحكم الخارجيه` | لوحة التحكم |

لا تشغل الأوامر من المجلد الرئيسي مباشرة. كل جزء يجب تشغيله من داخل المجلد الخاص به.

ابدأ دائما من المجلد الرئيسي للمشروع، وهو المجلد الذي يحتوي على هذه المجلدات:

```txt
backend
موقع اتعلم بل العربي
لوحه التحكم الخارجيه
```

مكان هذا المجلد قد يختلف من جهاز لآخر. المهم أن يفتح المبرمج Terminal داخل المجلد الرئيسي أولا، أو يدخل إليه بأمر `cd` حسب مكانه على جهازه.

مثال فقط، وليس مسارا ثابتا:

```bat
cd "D:\Projects\اسم-مجلد-المشروع"
```

## 1. تشغيل الباك اند Laravel

افتح نافذة CMD أو Terminal جديدة، ثم اكتب:

```bat
cd backend
php artisan serve --host=127.0.0.1 --port=5000
```

بعد التشغيل افتح هذا الرابط للتأكد أن الباك اند يعمل:

```txt
http://127.0.0.1:5000/api/health
```

## 2. تشغيل الموقع الرئيسي

افتح نافذة CMD أو Terminal ثانية، ثم اكتب:

```bat
cd "موقع اتعلم بل العربي"
npm run dev -- --host 127.0.0.1 --port 3000
```

بعد التشغيل افتح الموقع من:

```txt
http://127.0.0.1:3000/
```

## 3. تشغيل لوحة التحكم

افتح نافذة CMD أو Terminal ثالثة، ثم اكتب:

```bat
cd "لوحه التحكم الخارجيه"
npm run dev -- --host 127.0.0.1 --port 3001
```

بعد التشغيل افتح لوحة التحكم من:

```txt
http://127.0.0.1:3001/admin/
```

بيانات الدخول الافتراضية تأتي من `backend/.env`:

```txt
email: admin@et3alem.local
password: قيمة ADMIN_BOOTSTRAP_PASSWORD
```

مهم: غيّر `ADMIN_BOOTSTRAP_PASSWORD` و`VITE_ADMIN_PASSWORD` قبل تسليم المشروع أو نشره. القيم الموجودة في ملفات `.env.example` مجرد أمثلة وليست أسرارا جاهزة للإنتاج.

## روابط التشغيل النهائية

| الخدمة | الرابط |
|---|---|
| الموقع الرئيسي | `http://127.0.0.1:3000/` |
| لوحة التحكم | `http://127.0.0.1:3001/admin/` |
| اختبار الباك اند | `http://127.0.0.1:5000/api/health` |

## مشاكل شائعة وحلها

### مشكلة: Could not read package.json

مثال الخطأ:

```txt
npm error enoent Could not read package.json
```

السبب:

أنت تشغل أمر `npm run dev` من مجلد لا يحتوي على ملف `package.json`.

الحل:

ادخل إلى مجلد الموقع أو لوحة التحكم أولا، ثم شغل الأمر.

للموقع الرئيسي:

```bat
cd "موقع اتعلم بل العربي"
npm run dev -- --host 127.0.0.1 --port 3000
```

للوحة التحكم:

```bat
cd "لوحه التحكم الخارجيه"
npm run dev -- --host 127.0.0.1 --port 3001
```

### مشكلة: Could not open input file: artisan

مثال الخطأ:

```txt
Could not open input file: artisan
```

السبب:

أنت تشغل أمر Laravel من مجلد لا يحتوي على ملف `artisan`.

الحل:

ادخل إلى مجلد `backend` أولا:

```bat
cd backend
php artisan serve --host=127.0.0.1 --port=5000
```

### مشكلة: المنفذ مستخدم بالفعل

مثال الخطأ:

```txt
Port 3000 is already in use
```

الحل:

استخدم منفذا آخر.

مثال للموقع:

```bat
npm run dev -- --host 127.0.0.1 --port 3002
```

مثال للباك اند:

```bat
php artisan serve --host=127.0.0.1 --port=8000
```

### مشكلة: npm run dev لا يعمل بسبب نقص الملفات

لو ظهرت أخطاء بسبب نقص مكتبات Node.js، شغل هذا الأمر داخل مجلد الموقع أو لوحة التحكم:

```bat
npm install
```

ثم شغل المشروع مرة أخرى:

```bat
npm run dev
```

### مشكلة: Laravel لا يعمل بسبب نقص vendor

لو ظهر خطأ يدل أن مكتبات Laravel غير موجودة، شغل هذا الأمر داخل مجلد `backend`:

```bat
composer install
```

ثم شغل Laravel:

```bat
php artisan serve --host=127.0.0.1 --port=5000
```

### مشكلة: الموقع يعمل لكن البيانات لا تظهر

تأكد أولا أن الباك اند يعمل من هذا الرابط:

```txt
http://127.0.0.1:5000/api/health
```

لو الرابط لا يعمل، شغل Laravel من مجلد `backend`.

لو الرابط يعمل والموقع لا يعرض البيانات، أعد تشغيل الموقع الرئيسي ولوحة التحكم.

### مشكلة: ملفات الصور أو الصوت المرفوعة لا تظهر

في Laravel يجب ربط مجلد التخزين العام مرة واحدة:

```bat
cd backend
php artisan storage:link
```

لو كان `backend/public/storage` موجودا كمجلد عادي وليس رابطا، لا تحذفه قبل مراجعة محتواه حتى لا تفقد ملفات مرفوعة.

## ترتيب التشغيل الصحيح

يفضل تشغيل المشروع بهذا الترتيب:

1. شغل `backend` أولا.
2. شغل الموقع الرئيسي.
3. شغل لوحة التحكم.
4. افتح الروابط في المتصفح.

## ملاحظات سريعة

- استخدم 3 نوافذ CMD أو Terminal، واحدة لكل جزء.
- لا تغلق نافذة التشغيل أثناء استخدام المشروع.
- لو غيرت أي ملف إعدادات، أوقف التشغيل وشغله مرة أخرى.
- أهم شيء: كل أمر يجب أن يعمل من داخل المجلد الصحيح.
