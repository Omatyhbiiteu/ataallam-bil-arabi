# طريقة تشغيل المشروع لأي مبرمج

للتشغيل السريع في 3 ترمينالات استخدم أولاً ملف: [روابط فتح المشروع .md](./روابط%20فتح%20المشروع%20.md)

هذا المشروع مكوّن من 3 أجزاء:

- `backend`: Laravel API وقاعدة البيانات.
- `موقع اتعلم بل العربي`: واجهة المستخدم React/Vite.
- `لوحه التحكم الخارجيه`: لوحة التحكم React/Vite.

## المتطلبات

- PHP 8.2 أو أحدث
- Composer
- Node.js 20 أو أحدث
- MySQL/MariaDB أو قاعدة البيانات التي سيستخدمها المشروع

## 1. تشغيل Laravel

افتح Terminal من مجلد المشروع الرئيسي، ثم:

```powershell
cd backend
composer install
copy .env.example .env
php artisan key:generate
```

اضبط بيانات قاعدة البيانات داخل `backend/.env`.

لو تريد تسليم المشروع بدون بيانات، اعمل قاعدة بيانات فاضية باسم `et3alem_araby` ثم شغّل الجداول فقط:

```powershell
php artisan migrate
```

لا تشغّل seeders في هذا السيناريو، حتى لا تظهر بيانات أولية في الموقع. بعد أول تسجيل دخول للوحة التحكم، Laravel ينشئ حساب الأدمن الافتراضي تلقائيا إذا لم يكن موجودا.

لو تريد بيانات تجريبية للمنهج والقصص فقط:

```powershell
php artisan db:seed --class=SeedInitialCurriculumSeeder
php artisan db:seed --class=SeedInitialStoriesSeeder
```

لو تريد نفس بيانات جهازك، استورد ملف SQL الموجود في:

```text
database-export/et3alem_araby_clean_empty.sql
```

> ملاحظة: الملف الحالي نظيف وفارغ من بيانات المستخدمين. لو تم تصدير نسخة كاملة لاحقا، ضعها داخل `database-export` واذكر اسمها هنا بوضوح.

بعد تشغيل migrations، اربط ملفات الرفع العامة مرة واحدة:

```powershell
php artisan storage:link
```

لو ظهر أن `public/storage` موجود بالفعل، لا تمسحه مباشرة لأنه قد يحتوي ملفات مرفوعة. راجع محتواه أولا، ثم انقل الملفات المهمة إلى `storage/app/public` قبل إعادة إنشاء الرابط.

شغّل Laravel على أحد المنفذين التاليين:

```powershell
php artisan serve --host=127.0.0.1 --port=5000
```

أو:

```powershell
php artisan serve --host=127.0.0.1 --port=8000
```

اختبار سريع:

```powershell
curl http://127.0.0.1:5000/api/health
```

أو:

```powershell
curl http://127.0.0.1:8000/api/health
```

## 2. تشغيل موقع المستخدم

من Terminal جديد:

```powershell
cd "موقع اتعلم بل العربي"
npm install
copy .env.example .env
npm run dev
```

الرابط:

```text
http://127.0.0.1:3000/
```

## 3. تشغيل لوحة التحكم

من Terminal جديد:

```powershell
cd "لوحه التحكم الخارجيه"
npm install
copy .env.example .env
npm run dev
```

الرابط:

```text
http://127.0.0.1:3001/admin/
```

بيانات الأدمن الافتراضية تأتي من ملف `backend/.env`:

```text
email: admin@et3alem.local
password: قيمة ADMIN_BOOTSTRAP_PASSWORD
```

في قاعدة بيانات جديدة، Laravel ينشئ هذا الحساب تلقائيا عند أول محاولة دخول للوحة التحكم. غيّر `ADMIN_BOOTSTRAP_PASSWORD` و`VITE_ADMIN_PASSWORD` قبل تسليم المشروع أو نشره.

## طريقة اتصال React بالـ backend

في التطوير، اترك:

```env
VITE_BACKEND_API_URL=/api
```

لا تضع رابط Laravel الكامل في التطوير مثل `http://localhost:5000/api` إلا لو فاهم إعدادات CORS. وجود `/api` يجعل React يرسل الطلبات إلى Vite، وVite يمررها إلى Laravel.

ملفات `vite.config.ts` في الواجهتين تحاول الاتصال بالترتيب التالي:

1. القيمة الموجودة في `VITE_BACKEND_PROXY_URL`
2. `http://127.0.0.1:5000`
3. `http://127.0.0.1:8000`
4. `http://localhost:5000`
5. `http://localhost:8000`

لذلك لو مبرمج شغّل Laravel على 5000 أو 8000، الواجهات ستظل قادرة على الوصول إليه.

## حل مشاكل شائعة

لو ظهرت أخطاء API في React:

```text
Failed to fetch
تعذر الاتصال بالخادم
```

تأكد أن Laravel يعمل وأن هذا الرابط يرجع JSON:

```text
http://127.0.0.1:5000/api/health
```

أو:

```text
http://127.0.0.1:8000/api/health
```

لو ظهرت بيانات قديمة أو IDs غير موجودة مثل:

```text
No query results for model ... mod-en-1
```

غالبا المتصفح محتفظ ببيانات قديمة في `localStorage`. افتح DevTools ثم Console وشغّل:

```js
Object.keys(localStorage)
  .filter((key) => key.startsWith('hcard_'))
  .forEach((key) => localStorage.removeItem(key));
location.reload();
```

لو المشروع مطلوب يكون فاضي، لا تشغّل seeders. فقط امسح بيانات المتصفح القديمة وشغّل `php artisan migrate`.

لو المشروع مطلوب يكون فيه بيانات أولية، شغّل seeders المذكورة في خطوة Laravel.

## روابط التشغيل النهائية

- موقع المستخدم: `http://127.0.0.1:3000/`
- لوحة التحكم: `http://127.0.0.1:3001/admin/`
- اختبار Laravel: `http://127.0.0.1:5000/api/health` أو `http://127.0.0.1:8000/api/health`
